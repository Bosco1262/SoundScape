# SoundScape (声景)

[English](README.md) | [简体中文](README_zh-CN.md)

一个基于地理位置的地图应用，用户可以为特定地点提议和投票选择背景音乐 (BGM)。当用户进入一个已标记的区域时，应用会自动播放得票最高的音乐，创造一个属于大家的、存在于现实世界的声音景观。

## 项目概述

你是否曾走过一个公园，心里想着“如果这里能配上某首歌就完美了”？SoundScape 将这个想法变为现实。它是一场关于听觉与地理关联的社会实验，让社区能够共同决定他们物理环境的背景音乐。

通过一个基于地图的界面，你可以标记任何地点——一家咖啡店、一个风景优美的观景点、一条安静的街道——并推荐一首能够捕捉其精髓的歌曲。其他用户可以对这些建议进行投票。当你亲身进入一个拥有高票配乐的区域时，音乐会自动响起，让你沉浸在一个共享的、基于位置的声音景观中。

## 核心功能

-   **🗺️ 互动地图**：发现并探索由其他用户标记了音乐的地点。
-   **🎵 提议音乐**：在地图上标记任何地方，并为其氛围推荐一首合适的歌曲。
-   **👍 投票选曲**：通过对不同地点的音乐建议进行投票，来影响你所在地区的声音景观。
-   **🎧 自动播放**：通过地理围栏技术，应用会检测你何时进入标记区域，并自动播放获胜的配乐。
-   **🏆 排行榜**：查看你所在城市的热门歌曲，以及哪些用户是顶尖的“声景设计师”。
-   **👤 用户资料**：追踪你的提议、投票和你发现的地方。

## 技术栈

本项目将采用现代化的网络和移动技术构建：

-   **前端**: [React](https://reactjs.org/) 或 [React Native](https://reactnative.dev/)，以实现响应式和跨平台的用户体验。
-   **后端**: [Node.js](https://nodejs.org/) 与 [Express.js](https://expressjs.com/)，构建轻量级且可扩展的服务器。
-   **数据库**: 使用像 [MongoDB](https://www.mongodb.com/) 这样的 NoSQL 数据库来存储用户数据、地点和投票信息。
-   **地图与地理围栏**: 集成地图服务（如 Mapbox 或 Google Maps）和地理围栏 API，以处理位置追踪和触发事件。
-   **音乐服务**: 连接到音乐流媒体 API（如 Spotify 或 Apple Music），用于获取歌曲数据和播放。
